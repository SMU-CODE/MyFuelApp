# دليل تشغيل مشروع وقودي 

## متطلبات التشغيل الأساسية
- PHP 8.3.16
- MySQL 8.4.3
- Flutter SDK
- اتصال شبكة موحد

## 1. إعداد السيرفر (my_fuel_app_api)
 بعد اعدادات ال .env
1. افتح المشروع في VS Code
2. شغل Terminal ونفذ:
   ```
   composer install
   php artisan migrate:fresh --seed
   php artisan serve --host=0.0.0.0 --port=8000
   ```

ملاحظة: أمر `--host=0.0.0.0` يسمح بالوصول من أي جهاز على الشبكة المحلية

## 2. إعداد تطبيق الأندرويد (MyFuelApp)

1. افتح ملف الإعدادات:
   `lib/shared/api/ApiConstants.dart`

2. اضبط إعدادات الاتصال:
   - للمحاكي: `_currEnv = 2` (يستخدم 10.0.2.2)
   - للجوال الحقيقي: `_currEnv = 3` (يستخدم IP الكمبيوتر)

3. تأكد من تطابق IP في:
   - ملف ApiConstants.dart (_wifi)
   - ملف .env في السيرفر (APP_URL)

## 3. معرفة IP الكمبيوتر (للاتصال بالجوال)

1. على ويندوز:
   ```
   ipconfig
   ```
   ابحث عن IPv4 تحت اتصال الشبكة النشط

## 4. تشغيل التطبيق

1. في Terminal مشروع الفلاتر:
   ```
   flutter clean
   flutter pub get
   flutter run
   ```

## استكشاف الأخطاء الشائعة

1. إذا فشل الاتصال:
   - تأكد من تشغيل السيرفر
   - تأكد من تطابق IP في كلا المشروعين
   - جرب تبديل _currEnv بين 2 و 3

2. إذا ظهرت أخطاء في البيانات:
   ```
   php artisan migrate:fresh --seed
   ```

## ملاحظات تقنية هامة

1. سبب استخدام `--host=0.0.0.0`:
   - يسمح بالوصول من خارج الجهاز المضيف
   - ضروري لاختبار التطبيق على أجهزة حقيقية

2. لماذا نتجنب `copy .env.example .env`؟
   - في هذا الدليل نفترض وجود ملف .env مكون مسبقاً
   - يحتوي على الإعدادات الأساسية مثل APP_URL

3. تأكد من:
   - تطابق المنفذ (8000) في جميع الإعدادات
   - أن جميع الأجهزة على نفس الشبكة
   - تعطيل الجدار الناري للاختبارات الأولية
